# StyleSwift 代码重构示例

## 🔄 重构目标

1. **消除重复代码** - 提取公共函数到独立模块
2. **提高代码可维护性** - 统一代码风格和命名规范
3. **增强可读性** - 清晰的函数职责分离
4. **便于测试** - 模块化的函数更容易单元测试

## 📝 重构示例1：时间工具函数

### ❌ 重构前 - 在多个文件中重复的代码

#### `popup.js` 中的重复代码
```javascript
// 北京时间工具函数
function getBeijingTime() {
    const now = new Date();
    // 获取北京时间（UTC+8）
    const beijingTime = new Date(now.getTime() + (8 * 60 * 60 * 1000));
    return beijingTime;
}

function formatBeijingTime(date = null) {
    if (!date) date = getBeijingTime();
    return date.toISOString().replace('T', ' ').substring(0, 23) + ' CST';
}

function calculateDuration(startTime, endTime = null) {
    if (!endTime) endTime = new Date();
    return (endTime - startTime) / 1000; // 返回秒
}
```

## 📝 重构示例2：消息处理统一化

### ❌ 重构前 - 分散的消息处理

#### 在多个地方重复的消息发送逻辑
```javascript
// popup.js 中
chrome.tabs.sendMessage(tabId, {
    action: "applyStyle",
    style: css,
    styleId: styleId
}, function(response) {
    if (chrome.runtime.lastError) {
        console.error('发送消息失败:', chrome.runtime.lastError);
        reject(new Error('发送消息失败'));
        return;
    }
    if (response && response.success) {
        resolve();
    } else {
        reject(new Error('应用样式失败'));
    }
});

// 类似的代码在其他地方也有重复...
```

### ✅ 重构后 - 统一的消息处理模块

#### 1. 创建 `utils/messageUtils.js`
```javascript
/**
 * StyleSwift 消息处理工具函数
 */
class MessageUtils {
    /**
     * 安全发送消息到标签页
     * @param {number} tabId - 目标标签页ID
     * @param {Object} message - 消息对象
     * @param {Object} options - 选项配置
     * @returns {Promise} 返回Promise
     */
    static async sendMessageToTab(tabId, message, options = {}) {
        const { timeout = 10000, retries = 3 } = options;
        
        for (let attempt = 1; attempt <= retries; attempt++) {
            try {
                const response = await new Promise((resolve, reject) => {
                    const timeoutId = setTimeout(() => {
                        reject(new Error(`消息发送超时 (${timeout}ms)`));
                    }, timeout);
                    
                    chrome.tabs.sendMessage(tabId, message, (response) => {
                        clearTimeout(timeoutId);
                        
                        if (chrome.runtime.lastError) {
                            reject(new Error(chrome.runtime.lastError.message));
                            return;
                        }
                        
                        resolve(response);
                    });
                });
                
                return response;
            } catch (error) {
                console.warn(`消息发送尝试 ${attempt}/${retries} 失败:`, error);
                
                if (attempt === retries) {
                    throw error;
                }
                
                // 等待后重试
                await new Promise(resolve => setTimeout(resolve, 1000));
            }
        }
    }

    /**
     * 发送消息到背景脚本
     * @param {Object} message - 消息对象
     * @param {Object} options - 选项配置
     * @returns {Promise} 返回Promise
     */
    static async sendMessageToBackground(message, options = {}) {
        const { timeout = 30000 } = options;
        
        return new Promise((resolve, reject) => {
            const timeoutId = setTimeout(() => {
                reject(new Error(`背景脚本消息超时 (${timeout}ms)`));
            }, timeout);
            
            chrome.runtime.sendMessage(message, (response) => {
                clearTimeout(timeoutId);
                
                if (chrome.runtime.lastError) {
                    reject(new Error(chrome.runtime.lastError.message));
                    return;
                }
                
                resolve(response);
            });
        });
    }
}

// 导出到全局
window.StyleSwiftMessageUtils = MessageUtils;
```

#### 2. 重构后的代码使用
```javascript
// popup.js 中重构后的代码
async function applyCustomCSS(tabId, css, resolve, reject) {
    try {
        // 使用统一的消息发送工具
        const response = await StyleSwiftMessageUtils.sendMessageToTab(
            tabId, 
            {
                action: "applyStyle",
                style: css,
                styleId: styleId
            },
            {
                timeout: 5000,
                retries: 2
            }
        );
        
        if (response && response.success) {
            resolve();
        } else {
            throw new Error('应用样式失败');
        }
    } catch (error) {
        console.error('应用自定义CSS失败:', error);
        reject(error);
    }
}
```

## 💡 重构最佳实践

### 1. **渐进式重构**
- 不要一次性重构所有代码
- 先提取工具函数，再重构使用方
- 保持功能不变的情况下改进代码

### 2. **充分测试**
- 重构前后功能保持一致
- 重点测试边界情况
- 确保错误处理正常工作

### 3. **文档同步更新**
- 更新代码注释
- 更新使用说明
- 保持文档与代码同步

### 4. **团队协作**
- 重构前与团队成员沟通
- 制定统一的编码规范
